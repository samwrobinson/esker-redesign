# Repurpose CodeStitch Astro Template for a New Client

## Context
I run Esker Designs. I build websites for small local businesses by repurposing a
CodeStitch Astro template I already own. I have a working version of this template
(components, CSS using CodeStitch variables, and JS already wired up). For each new
client I keep the structure and styling intact and swap in their content, branding,
and photos so the result feels custom — not templated.

I'm giving you a new client's info and photos below. Produce their version of the site.

## The Client
- Business name: [NAME]
- Industry / what they do: [e.g. roofing contractor, therapist, café]
- Location(s) they serve: [CITY, STATE / service area]
- Services to feature: [list 3–6]
- Phone / email / address: [CONTACT INFO]
- Hours (if relevant): [HOURS]
- Brand vibe: [e.g. rugged & bold / warm & trustworthy / modern & minimal]
- Brand colors (if they have them): [HEX or "use template defaults / pick to match logo"]
- Anything to emphasize: [reviews, awards, "family-owned since X", guarantees, etc.]
- Photos: attached / in the uploads folder. [Note any "this one is the hero," "these
  are gallery," "this is the logo," etc. — otherwise sort them by what makes sense.]

## What to do
1. Work from my existing template structure. Keep every component, the CSS
   architecture, the CodeStitch CSS variables, the JS, and all responsive behavior
   intact. This is a content + branding swap, not a rebuild.
2. Replace all copy: headlines, subheads, body text, CTAs, nav labels, footer.
   Write it in the client's voice and tone for their [brand vibe]. No lorem ipsum,
   no leftover placeholder text from the template or a previous client.
3. Update branding: set the brand color variables to the client's colors (or pick
   ones that suit their logo/industry if none given). Swap the logo and business name
   everywhere it appears.
4. Sort and place the photos sensibly: best wide shot → hero, service/work shots →
   relevant sections and gallery, headshot/team → about. Add descriptive alt text
   for every image (good for SEO and accessibility).
5. Update contact info, map/address, phone links (tel:), email, hours, and the
   contact form labels to match the client.
6. SEO: rewrite the <title> and meta description per page around the client's
   business, location, and services. Update any schema/structured data with the
   client's name, address, and phone.
7. Scrub for leftovers: no old client names, old phone numbers, old alt text, or
   template demo content anywhere.

## What NOT to touch
Don't restructure the CSS, rename classes, rip out the JS, or change the layout/
component system unless I explicitly ask. If something genuinely needs a structural
change to fit this client, flag it and ask first rather than improvising.

## Output
Give me the updated files. If you change multiple files, tell me which ones and
summarize what changed in each so I can review before I build and deploy.
